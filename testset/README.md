# 垃圾厢房治理事件测试集

这个目录用于组织客户场景测试集。测试目标不是“垃圾分类”，而是固定点位下的治理事件识别与误报压测。

## 收集原则

- 每个事件至少 20 张可用样本，目标 50 张以上。
- 每个事件必须包含正例、负例、边界例。
- 每张图必须记录来源、许可/可用性、采集日期、是否可训练、是否可公开展示。
- 有框标注的数据可用于目标检测训练；只有图像级标签的数据只用于状态/事件识别，不在页面上展示检测框。
- 客户页面只展示真实模型输出；未验证样本只能进入测试集或复核队列。

## 事件类别

1. 小包垃圾落地
   - 正例：袋装垃圾、零散垃圾、远景小目标、遮挡、贴边。
   - 负例：桶体、路沿、车辆、树叶、反光、检测区域外垃圾。
   - 标注建议：垃圾袋/零散垃圾框 + 所属区域。

2. 地面脏污
   - 正例：油污、泥渍、积水、污渍拖痕。
   - 负例：雨后反光、地砖纹理、阴影、正常湿地面。
   - 标注建议：图像级状态优先，必要时加区域 mask；重点测误报。

3. 垃圾满溢/堆积
   - 正例：桶口外溢、箱房内堆放、垃圾桶旁堆积。
   - 负例：正常满载但未外溢、空桶、背景杂物。
   - 标注建议：状态标签 + 桶口/堆积区域框。

4. 箱盖被压/阻挡
   - 正例：大件物体压盖、袋子卡住箱盖、投放口被挡。
   - 负例：正常闭合、正常打开、人手短暂接触。
   - 标注建议：阻挡物框 + 箱盖状态。

5. 违规投放行为
   - 正例：非投放时间拎袋进入、随地丢弃、弯腰/投掷、人和垃圾袋交互。
   - 负例：路过、保洁作业、正常投放、取物。
   - 标注建议：视频片段优先，记录前后帧、人、袋、投放区域和时间条件。

6. 保洁/工作人员场景
   - 正例：人在岗、清扫动作、清运动作、工作服/RFID/人脸待接入。
   - 负例：普通居民、路人、快递/车辆经过。
   - 标注建议：人和工具框 + 行为状态。

7. 复杂环境
   - 正例：低光、雨雾、强反光、夜间、角度差、多人/车辆/设备干扰。
   - 负例：清晰白天固定机位。
   - 标注建议：作为所有事件的环境标签，而不是单独业务告警。

## 文件说明

- `taxonomy.json`：事件类别、正负边界例和标注要求。
- `source_candidates.json`：可追溯公开数据源候选。
- `manifest_template.csv`：单张图片/视频帧的登记模板。
- `manifest.csv`：当前 seed 样本候选清单。
- `build_seed_manifest.py`：从已有真实数据和 TACO 候选源生成 seed manifest。

## 当前 seed manifest

已生成 100 条候选样本：

- 垃圾满溢/堆积：50 条，来自 `UniqueData/outdoor_garbage`，图像级状态标签。
- 小包垃圾落地：50 条，来自 TACO，带实例标注候选，需要人工筛选固定点位相似度。

其余事件已有来源候选和字段标准，未纳入本批 seed manifest。

## 当前数据边界

当前已训练 demo 模型使用 `UniqueData/outdoor_garbage`，可覆盖“满溢/未满载/周边散落”的图像级状态识别。它不能覆盖完整治理事件，也不能直接支持 YOLO 框评估。下一轮应优先补齐小包落地、地面脏污、箱盖阻挡和违规投放行为数据。
